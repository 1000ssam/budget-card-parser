# 사업관리카드 파서 - 블로그 소재 정리

## 한 줄 요약

학교 예산 사업관리카드 엑셀 파일의 들여쓰기로 병합된 계층 구조를 자동 파싱하여, 세부사업/세부항목/원가통계비목이 분리된 정규화 테이블로 변환해주는 웹앱.

## 이 도구가 필요한 상황

- 사업관리카드(예산) 엑셀 파일에서 "세부사업 > 세부항목 > 원가통계비목"이 들여쓰기(선행 공백)로 하나의 A열에 병합되어 있어, 엑셀 필터/피벗/VLOOKUP 등을 쓸 수 없는 상황
- 수작업으로 계층 구조를 분리하기엔 행이 많고 반복 작업이 잦은 상황
- 파싱된 데이터를 엑셀에 바로 붙여넣거나 xlsx로 다운로드해서 후속 작업에 쓰고 싶은 상황

## 핵심 기능

| 기능 | 설명 |
|------|------|
| 파일 업로드 | 드래그 앤 드롭 또는 클릭으로 .xls/.xlsx 업로드 |
| 자동 파싱 | 헤더 행 자동 감지 + 들여쓰기 기반 계층 구조 파싱 |
| 정규화 테이블 | 세부사업, 세부항목, 원가통계비목을 별도 컬럼으로 분리하여 테이블 표시 |
| 행 선택 | 체크박스로 개별/전체 선택 (indeterminate 상태 지원) |
| 클립보드 복사 | TSV + HTML 테이블 동시 복사 (엑셀, 노션 붙여넣기 호환) |
| xlsx 다운로드 | 정규화된 데이터를 xlsx 파일로 다운로드 (컬럼 너비 자동 조정) |
| 헤더 포함 옵션 | 복사 시 헤더 포함/미포함 선택 가능 (기본값: 미포함) |
| 선택 항목 내보내기 | 전체 또는 선택된 행만 복사/다운로드 |

## 사용 흐름

1. 사업관리카드 .xls 또는 .xlsx 파일을 드래그하거나 선택하여 업로드
2. 파일이 자동으로 파싱되어 정규화된 테이블로 화면에 표시
3. 체크박스로 원하는 행을 선택 (또는 전체 선택)
4. "전체 다운로드/복사" 또는 "선택 다운로드/복사" 버튼으로 데이터 내보내기
5. 엑셀이나 노션에 붙여넣기하여 후속 작업 수행

## 기술적 특징

- **100% 클라이언트 사이드 처리**: 파일이 서버로 전송되지 않음. FileReader API로 브라우저에서 직접 파싱. 서버/DB 불필요.
- **헤더 행 자동 감지**: "세부사업", "세부항목", "원가통계비목" 텍스트가 포함된 셀을 탐색하여 헤더 위치를 자동 결정. 2행 병합 헤더 구조 지원.
- **들여쓰기 레벨 자동 판별**: A열의 모든 선행 공백 수를 수집 → 유니크 값 정렬 → 인덱스를 레벨로 매핑. 레벨 0(합계)은 스킵, 레벨 1~3이 각각 세부사업/세부항목/원가통계비목에 대응.
- **컨텍스트 상속**: 상위 레벨 값을 state로 유지하면서 하위 행에 자동 상속. 레벨이 바뀌면 하위 컨텍스트 초기화.
- **듀얼 포맷 클립보드 복사**: ClipboardItem API로 text/plain(TSV)과 text/html(HTML 테이블)을 동시에 클립보드에 기록. 엑셀에 붙여넣으면 TSV로, 노션에 붙여넣으면 HTML 테이블로 인식됨.
- **OG 이미지 동적 생성**: Next.js ImageResponse API (Edge Runtime)로 opengraph-image.tsx에서 1200x630 PNG를 코드로 생성. 별도 이미지 파일 불필요.
- **Turbopack 개발 서버**: `next dev --turbopack`으로 빠른 HMR 지원.

## 디자인

- **폰트**: Pretendard Variable (CDN)
- **액센트 컬러**: `#D2886F` (테라코타 계열)
- **색상 체계**: CSS 변수 기반. `--accent`, `--bg-white`(#ffffff), `--bg-subtle`(#fafafa), `--text-main`(#171717), `--text-sub`(#525252), `--text-light`(#a3a3a3), `--border-color`(#e5e5e5)
- **다크 모드**: CSS `prefers-color-scheme: dark` 미디어 쿼리로 변수 오버라이드 정의됨
- **레이아웃**: 최대 너비 7xl, 중앙 정렬. 콤팩트한 패딩(py-6 px-4)
- **테이블**: 고정 헤더(sticky), 최대 높이 500px 스크롤, 선택 행 하이라이트(`bg-[#D2886F]/10`)
- **애니메이션**: slide-up (opacity + translateY, 0.2s ease-out)
- **UI 전체 한글화**

## 제한사항 / 참고

- 헤더 행에 "세부사업", "세부항목", "원가통계비목" 중 하나가 반드시 포함되어 있어야 함
- 들여쓰기(선행 공백)로만 계층 구조를 판단함. 다른 방식(셀 병합, 색상 구분 등)은 미지원
- 2행 병합 헤더만 지원 (3행 이상 미지원)
- 대용량 파일(1만 행 이상) 시 브라우저 성능 저하 가능 (모든 파싱이 클라이언트 사이드)
- IE 미지원 (최신 Chrome, Safari, Firefox, Edge 지원)
- 서버 사이드 파싱 없음. 파일이 외부로 전송되지 않으므로 보안 측면에서는 장점
- 현재 버전: v1.1.0

## 대상 사용자

한국 초/중/고등학교에서 예산을 관리하는 교사. 사업관리카드 엑셀 파일을 다루며, 들여쓰기로 병합된 데이터를 정리해야 하는 실무자.

## 만든 이유 (동기)

학교 예산 사업관리카드는 "세부사업 > 세부항목 > 원가통계비목"이 들여쓰기(공백)로 하나의 열에 계층적으로 표현되어 있다. 이 구조는 사람 눈으로 읽기에는 괜찮지만, 엑셀에서 필터/정렬/피벗/수식을 적용하려면 각 계층이 별도 컬럼으로 분리되어야 한다. 매번 수작업으로 분리하는 반복 노동을 자동화하기 위해 만들었다.
